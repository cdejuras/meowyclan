# Contributing and Project Rules

Follow the repository's Additional Rules and Flow (see `context/Additional-Rules-and-Flow.txt`). Key points:

- Create unit tests for new code.
- Add/update `CHANGELOG.md` for features and changes.
- Add a one-paragraph feature summary for each new feature under `docs/feature-summaries/`.
- Always open a feature issue and get explicit confirmation before starting work. Use `.github/ISSUE_TEMPLATE/feature_request.md`.

Formatting and conventions:
- Java: use JDK 21, Spring Boot conventions, Lombok present in backend module.
- Frontend: use Vite + TypeScript, follow React patterns in `src/components`.
